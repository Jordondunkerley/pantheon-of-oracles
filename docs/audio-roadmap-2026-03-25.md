# Pantheon of Oracles – Audio Roadmap (2026-03-25)

## Principle
Strengthen what works first, then expand step by step.

Audio communication is the next major experiential leap after text interaction.
Video should be treated as a later extension built on the same oracle core.

## Recommended phased path

### Phase 1 – Text-first foundation
- stable oracle identities
- session persistence
- provider binding
- import/export consistency

### Phase 2 – Oracle voice output
- assign voice profile metadata per oracle
- generate spoken oracle responses from text
- support per-oracle voice flavor and tone

### Phase 3 – Voice input
- speech-to-text capture from user
- route spoken input into oracle sessions
- preserve transcripts in session history

### Phase 4 – Conversational audio loop
- text + voice output + voice input in one flow
- oracle audio presence feels natural and persistent

### Phase 5 – Visual / video embodiment
- avatar presence
- lip-sync / animation / cinematic responses
- richer visual oracle communication

## Core rule
Audio and later video must still originate from the same canonical oracle identity system used by the desktop core and future games.

## Immediate implication for the prototype
Add and normalize fields such as:
- voice style
- preferred voice profile
- audio readiness
- future avatar/video readiness hooks
