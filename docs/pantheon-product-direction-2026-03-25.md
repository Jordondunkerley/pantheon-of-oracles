# Pantheon of Oracles Product Direction (2026-03-25)

## Current decision
The desktop app should no longer be framed primarily as a Clawdbot operations console.

It should now be developed and presented as a **Pantheon of Oracles-first marketable desktop product**.

## Product framing
Pantheon of Oracles is the primary brand and value proposition.
The software should help users:
- create their pantheon
- load astrology data
- generate and manage oracle companions
- connect a preferred LLM/backend
- interact with those oracles as persistent agents

## Immediate product priorities
1. Polished onboarding flow
2. Refined player/oracle/astrology schemas
3. Model-agnostic provider setup
4. Rich oracle workspace
5. Desktop product polish and packaging

## Secondary / deferred systems
- full multiplayer execution
- raids / dungeons / warbands
- throne worlds / descendants / leviathans
- visual creature systems

These should remain represented as future hooks where helpful, but should not distort the first marketable prototype.

## Strategic note
This first desktop product should become the foundation for:
- later mobile app versions
- expanded live-service systems
- deeper multiplayer and story systems
- broader oracle-as-agent deployment across platforms
