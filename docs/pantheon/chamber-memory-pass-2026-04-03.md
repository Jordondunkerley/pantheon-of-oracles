# Chamber Memory Pass

## Why this pass mattered
The chamber needed a more explicit world-memory layer so the player feels that prior openings and crossings remain meaningful.

## What changed

- tied chamber memory language to unlocked-scene count
- reinforced that the chamber world should feel changed by what has already been opened

## Product effect
This makes the chamber feel less like a resettable interface and more like a place that retains the player’s passage.
