# Scene-Based Redesign

## Why this pass mattered
The user explicitly wants the product to feel more like a video game and less like a scheduler/dashboard or sprawling one-page prototype.

## What changed
- shifted from long-page multi-panel sprawl toward scene-based progression
- introduced scene flow:
  - title screen
  - first entrance
  - chart awakening
  - council formation
  - chamber hub
  - oracle room
- added progression state in app data
- made oracle selection push the user into the Oracle Room scene

## Product effect
This is the first real architectural move from dashboard UX toward chamber/game UX.

## Next likely priorities
1. transition polish / animations / reveal moments
2. stronger step-gating and unlock logic
3. reduce lingering admin-surface feel inside scenes
4. ship updated repo checkpoint once validated
